# 参考拆书风格综合指南

## 目标

从项目的 `参考拆书/` 目录中提取文笔风格信息，综合分析多本参考书的写作特点，整合成适合当前项目的风格指南，输出到 `设定集/参考拆书综合风格指南.md`。

## 执行原则

1. **多源综合**：读取所有拆书的文笔风格文件（`03_文笔风格.md`），提取共同点和特色
2. **可操作性**：将抽象的风格分析转化为具体可执行的写作规则
3. **项目适配**：结合当前项目的题材、基调、目标读者，调整风格建议
4. **证据驱动**：所有风格建议都要有来自拆书的具体证据支撑

## 输入

- 项目根目录（包含 `参考拆书/` 和 `设定集/` 目录）
- 项目类型（历史/官场/玄幻/都市等，从 `state.json` 或 `大纲/总纲.md` 推断）

## 输出

- `设定集/参考拆书综合风格指南.md`：包含以下章节
  - 参考书目清单
  - 共性风格特征（句长、段长、对白、节奏等）
  - 分类风格特征（按题材/类型分组）
  - 可复用模式库
  - 避坑指南（常见问题）
  - 项目适配建议

## 执行流程

### Step 1：收集拆书信息

扫描 `参考拆书/` 目录，收集：
- 所有拆书目录名称（作为参考书目）
- 每个拆书的 `03_文笔风格.md` 文件内容
- 每个拆书的 `00_总览.md` 文件内容（了解题材和类型）

### Step 2：提取风格特征

从每个文笔风格文件中提取：
1. **句式特征**：平均句长、短句比例、句式模式
2. **段落特征**：平均段长、段落结构、换行习惯
3. **对白特征**：对白占比、对白功能、对白风格
4. **描写特征**：动作/心理/说明占比、描写手法
5. **节奏特征**：推进方式、悬念设置、高潮处理

### Step 3：综合分析

1. **共性特征**：找出多本拆书共有的写作特点
2. **分组特征**：按题材/类型分组（如历史类、军事类、官场类）
3. **极端案例**：找出最特殊、最鲜明的风格样本

### Step 4：生成指南

按以下结构生成风格指南：

```markdown
# 参考拆书综合风格指南

## 参考书目

列出所有参考的拆书名称和基本信息

## 共性风格特征

### 句式与段落
- 基于多本拆书统计的句长/段长范围
- 常见的句式模式
- 段落结构特点

### 对白特点
- 对白占比范围
- 对白功能分布
- 对白风格特征

### 描写特点
- 动作/心理/说明的典型比例
- 常用描写手法
- 描写节奏特点

## 分类风格特征

### 历史官场类
- 基于具体拆书的特征分析
- 可复用模式

### 军事战争类
- 基于具体拆书的特征分析
- 可复用模式

### ...（其他类型）

## 可复用模式库

### 开头模式
- 从拆书中提取的优秀开头案例

### 对话模式
- 精彩对话案例

### 动作描写模式
- 精彩动作描写案例

### ...（其他模式）

## 避坑指南

- 拆书中发现的问题和避免方法

## 项目适配建议

- 结合当前项目的具体建议
- 推荐重点参考的拆书
- 需要规避的风格问题
```

## 使用方法

在项目根目录执行：

```bash
/webnovel-style-synth
```

或指定拆书目录：

```bash
/webnovel-style-synth --source 参考拆书
```

执行完成后检查：

- [ ] 可复用模式有具体案例
