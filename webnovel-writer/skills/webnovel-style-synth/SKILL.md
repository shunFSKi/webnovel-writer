# 参考拆书风格综合指南

## 目标

从项目的 `参考拆书/` 目录中提取文笔风格信息，综合分析多本参考书的写作特点，整合成适合当前项目的风格指南，输出到 `设定集/参考拆书综合风格指南.md`。

**新增功能**：支持策略同步，将分析结果同步到项目风格文件中。

## 执行原则

1. **多源综合**：读取所有拆书的所有分析文件（00-04），提取共同点和特色
2. **可操作性**：将抽象的风格分析转化为具体可执行的写作规则
3. **项目适配**：结合当前项目的题材、基调、目标读者，调整风格建议
4. **证据驱动**：所有风格建议都要有来自拆书的具体证据支撑
5. **策略可选**：支持 append/merge/replace 三种同步策略

## 输入

- 项目根目录（包含 `参考拆书/` 和 `设定集/` 目录）
- 项目类型（历史/官场/玄幻/都市等，从 `state.json` 或 `大纲/总纲.md` 推断）

## 输出

- `设定集/参考拆书综合风格指南.md`：包含以下章节
  - 参考书目清单
  - 共性风格特征（句长、段长、对白、节奏等）
  - 分类风格特征（按题材/类型分组）
  - 可复用模式库
  - 避坑指南（常见问题）
  - 项目适配建议

## 执行流程

### Step 1：收集拆书信息

扫描 `参考拆书/` 目录，收集：
- 所有拆书目录名称（作为参考书目）
- 每个拆书的 5 个分析文件：
  - `00_总览.md` - 类型、题材、目标读者
  - `01_剧情结构.md` - 开篇钩子、节奏模式
  - `02_人物特点.md` - 主角特点、对话风格
  - `03_文笔风格.md` - 句长段长、对白占比
  - `04_常用词句.md` - 高频动词、句式模板

### Step 2：提取风格特征

从所有分析文件中提取：
1. **题材类型**：类型分布、题材分布
2. **开篇钩子**：钩子类型、使用频率
3. **节奏模式**：螺旋上升、三幕式、闭环式
4. **句式特征**：平均句长、短句比例、句式模式
5. **段落特征**：平均段长、段落结构、换行习惯
6. **对白特征**：对白占比、对白功能、对白风格
7. **描写特征**：动作/心理/说明占比、描写手法
8. **常用词句**：高频动词、意象、句式模板

### Step 3：综合分析

1. **共性特征**：找出多本拆书共有的写作特点
2. **分组特征**：按题材/类型分组（如历史类、军事类、官场类）
3. **极端案例**：找出最特殊、最鲜明的风格样本

### Step 4：生成指南

按以下结构生成风格指南：

```markdown
# 参考拆书综合风格指南

## 参考书目
列出所有参考的拆书名称和基本信息

## 题材与类型分布
类型分布、题材分布统计

## 开篇钩子分析
常见钩子类型及频率

## 节奏与结构特点
节奏模式、结构特点分析

## 共性风格特征
### 句式与段落
### 对白特点
### 描写特点
### 句式与用词

## 各书详细特征
### 剧情结构
### 人物特点
### 文笔风格
### 常用词句

## 可复用模式建议
开头模式、对话模式、动作描写模式、节奏模式

## 避坑指南
常见问题和避免方法

## 项目适配建议
结合当前项目的具体建议
```

### Step 5（可选）：策略同步

如果指定了 `--strategy` 和 `--target`，将分析结果同步到项目风格文件：

- **preferences**: 同步到 `.webnovel/preferences.json`
- **style**: 同步到 `设定集/写作风格.md`
- **contract**: 同步到 `设定集/风格契约.md`
- **all**: 同步到所有文件

## 使用方法

### 基础用法（仅生成风格指南）

在项目根目录执行：

```bash
/webnovel-style-synth
```

或指定拆书目录：

```bash
/webnovel-style-synth --source 参考拆书
```

### 策略同步用法（生成指南 + 同步到项目文件）

#### 1. 追加模式（append）

在现有内容后添加新内容：

```bash
# 追加到写作风格
/webnovel-style-synth --strategy append --target style

# 追加到所有文件
/webnovel-style-synth --strategy append --target all
```

**适用场景**：
- 保留现有风格，补充参考拆书的新发现
- 项目已有成熟风格，只是参考拆书做补充

#### 2. 合并模式（merge）

智能合并现有和新内容：

```bash
# 合并到 preferences.json
/webnovel-style-synth --strategy merge --target preferences

# 合并到所有文件
/webnovel-style-synth --strategy merge --target all
```

**适用场景**：
- 项目风格还在发展中，需要融合参考拆书特点
- 保留现有有效配置，补充新发现

#### 3. 替换模式（replace）

完全替换特定章节：

```bash
# 替换风格契约
/webnovel-style-synth --strategy replace --target contract

# 替换所有文件
/webnovel-style-synth --strategy replace --target all
```

**适用场景**：
- 项目风格需要重构
- 完全按照参考拆书建立新风格

### 策略对比

| 策略 | preferences.json | 写作风格.md | 风格契约.md | 适用场景 |
|------|------------------|------------|------------|----------|
| append | 保留现有，追加新字段 | 在文件末尾追加 | 在文件末尾追加 | 保留现有，做补充 |
| merge | 智能合并字段 | 保留现有，追加章节 | 保留现有，追加章节 | 融合发展 |
| replace | 完全替换风格配置 | 完全替换文件 | 完全替换文件 | 风格重构 |

### 目标文件说明

#### preferences.json (`.webnovel/preferences.json`)
- **作用**：运行时风格偏好配置
- **格式**：JSON
- **内容**：tone、style、avoid、focus、examples 等
- **优先级**：项目本地最高优先级

#### 写作风格.md (`设定集/写作风格.md`)
- **作用**：完整的写作风格文档
- **格式**：Markdown
- **内容**：详细的风格指南、示例、检查清单
- **优先级**：项目风格权威文档

#### 风格契约.md (`设定集/风格契约.md`)
- **作用**：运行时风格镜像，快速调用
- **格式**：Markdown
- **内容**：一句话口令、必守原则、硬约束、检查清单
- **优先级**：与写作风格.md 配合使用
