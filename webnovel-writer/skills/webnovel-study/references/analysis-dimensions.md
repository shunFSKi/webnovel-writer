# Analysis Dimensions

## 1. 总目标

分析不是复述剧情，而是回答：
- 这本书怎么抓住读者
- 靠谁抓住读者
- 靠什么节奏一直推下去
- 哪些打法适合迁移

**核心问题：它为什么能签约/火，我能学到什么？**

## 2. 输入优先级

分析时优先使用：
1. `study_meta.json`
2. `chapter_index.json`
3. `chapter_source.jsonl`
4. 必要时再参考原书简介或抽样正文

原则：
- 逐章分析以缓存为准，不再重复手动切章
- `confidence` 与 `evidence_basis` 应继承源数据边界

## 3. 逐章分析维度

每章优先回答：
- 这一章解决了什么主问题
- 给了什么反馈
- 抛了什么新问题
- 主角是主动还是被动
- 这一章属于哪个剧情功能

### plot_role 枚举建议
- `开局`
- `承接`
- `升级`
- `对抗`
- `反转`
- `回收`
- `过渡`
- `高潮`
- `低谷`

### hook_types 常见值
- `事故钩`
- `悬念钩`
- `身份钩`
- `关系钩`
- `任务钩`
- `危险钩`
- `真相钩`
- `反差钩`

### payoffs 常见值
- `打脸`
- `资源到手`
- `身份抬升`
- `关系推进`
- `危机暂解`
- `新线索`
- `能力兑现`
- `读者预期兑现`

## 4. `chapter_analysis.jsonl` 最小字段

```json
{
  "chapter_number": 1,
  "chapter_title": "第1章 暴雨夜",
  "summary": "主角在险情初发时被迫接任务，并靠职业判断先做第一轮止损。",
  "plot_role": "开局",
  "hook_types": ["事故钩", "危险钩"],
  "payoffs": ["能力兑现", "危机暂解"],
  "new_question": "这场险情究竟是天灾还是人祸？",
  "characters_entered": ["主角", "直属上官"],
  "conflict_level": 4,
  "style_signals": ["短段快切", "对白带信息"],
  "notes": "基于标题与可读片段推断。",
  "confidence": "medium",
  "evidence_basis": ["title", "excerpt"]
}
```

要求：
- `summary` 控制在 50-120 字
- `style_signals` 只写抽象特征，不抄长句
- `notes` 要诚实说明是否为结构推断

## 5. 剧情结构分析维度

必须看：
- 标题承诺在几章内兑现
- 前三章是否形成小闭环
- 主线是否持续可见
- 中段是否出现失速
- 卷末是否有新钩子接下一段

建议拆成：
- 开篇抓读
- 中段续航
- 后段加速
- 结尾/卷末挂钩

## 6. 人物分析维度

### 主角
- 卖点
- 欲望
- 缺陷
- 先手优势
- 决策模型
- 爽点来源

### 配角
- 功能分工
- 是否能补主角短板
- 是否有单独辨识度
- 是否只在需要时出现

### 反派
- 分层
- 对主角构成哪种压力
- 是否只会提供脸谱化阻力
- 是否形成递进升级

## 7. 节奏分析维度

重点看：
- 每章是否解决一个主问题
- 每章是否留下继续读的理由
- 强反馈是不是来得早
- 过渡章是否也有小推进
- 是否出现连续多章同质化

## 8. 当前项目借鉴分析维度

在 `--compare-current` 下，必须比较：
- 标题承诺兑现速度
- 前三章抓读强度
- 主角主动性
- 爽点反馈速度
- 反派压迫感构建
- 章末留钩具体度
- 每次成果是否换到更大权限

## 9. 退化模式写法要求

### `full_text`
- 可以按正常精读口径输出
- `confidence` 可高于结构推断

### `degraded_structure_only`
- 剧情和节奏仍可正常分析
- 人物/风格/词句必须标 `medium` 或 `low`
- 明确写“基于标题序列、简介、抽样片段推断”

### `blocked`
- 不写逐章分析，不生成完整报告
- 只输出阻断原因

## 10. 不合格分析示例

❌ 不合格：
- 第5章主要讲主角去了哪里，遇到了谁……

✅ 合格：
- 第5章的功能是”把前面埋下的危险转成公开冲突”，主角第一次从被动挨打转成主动抢先手，因此读者会觉得书开始真正起飞。

## 11. 六维度分析框架（增强版）

### 一、开篇钩子分析（重点拆前三章）
**核心问题：读者读完前三章会不会继续看？为什么？**

必答：
1. 第一章第一段在做什么？
   - 是先交代背景还是直接进事件？
   - 读者读完第一段知道了什么信息？
   - 有没有悬念或冲突？

2. 前三章的核心功能分别是什么？
   - 第一章完成了什么任务？
   - 第二章完成了什么任务？
   - 第三章完成了什么任务？

3. 吸引读者继续看的钩子是什么？
   - 如果你要弃书，会在哪里弃？为什么？

4. 信息投放节奏
   - 前三章给了读者多少信息？
   - 哪些信息是立刻告诉读者的？
   - 哪些信息是藏着的（制造悬念）？
   - 信息投放的密度是不是合理？

### 二、爽点结构分析（新增核心维度）
**核心问题：爽点是怎么铺垫和释放的？**

必答：
1. 标记出每一个爽点的位置（第几章第几段）
2. 每个爽点的类型是什么？
   - 打脸/碾压/获得宝物/升级/身份揭示/逆转局面/智商碾压

3. 每个爽点之前有多少章的铺垫/压制？
   - 压制的具体方式是什么？
   - 压制到什么程度？

4. 爽点释放的方式是什么？
   - 是一次性释放还是分层释放？
   - 释放之后有没有收获/反馈环节？

5. 总结这本书的爽点循环模式
   - 小循环大概几章一轮？
   - 大循环大概几章一轮？
   - 这个节奏给你什么感觉？太快/太慢/刚好？

### 三、人物塑造分析
**核心问题：人物有没有辨识度？对话能不能遮住名字分清？**

必答：
1. 主角
   - 第一次出场时读者对他的印象是什么？
   - 作者用了什么手段让读者记住他？（一个动作/一句话/一个决定）
   - 他的核心性格是什么？是通过什么展示的？
   - 他说话有没有辨识度？举例。

2. 配角/反派
   - 写得最好的配角是谁？好在哪里？
   - 反派是怎么出场的？第一次出场是不是就让人讨厌？
   - 反派有没有”人味”？还是纯工具人？

3. 对话差异
   - 不同角色说话有没有区别？
   - 能不能遮住名字分辨出是谁在说话？
   - 举出两段对话，说明角色语言差异在哪里

### 四、情绪设计分析（增强）
**核心问题：情绪是怎么制造的？直接写还是情境带？**

必答：
1. 标记出你读的时候产生情绪波动的位置
   - 什么地方让你觉得爽？
   - 什么地方让你紧张？
   - 什么地方让你觉得虐/心酸？
   - 什么地方你想跳过？（说明那段写得差）

2. 情绪是怎么制造的？
   - 是靠直接描写情绪（”他很愤怒”）？
   - 还是靠情境制造情绪（场景、行为、对话、细节）？
   - 举例说明最好的一段情绪设计

3. 章末处理
   - 抽取5个章节的结尾
   - 每个结尾用的是什么类型的钩子？
     - 悬念/反转/情绪高点/信息差/卡在高潮
   - 哪个结尾写得最好？为什么？

4. 情绪密度
   - 平均每章有几次情绪起伏？
   - 有没有整章情绪平淡的？在第几章？
   - 情绪密度最高的是哪一章？

### 五、节奏和叙事分析
**核心问题：对话与叙述比例？世界观怎么告诉读者？**

必答：
1. 对话与叙述的比例大概是多少？
2. 有没有大段的环境描写或心理描写？出现在什么位置？
3. 信息传递的方式
   - 世界观设定是怎么告诉读者的？
   - 是旁白直接说？还是通过剧情和对话自然带出？
4. 场景转换
   - 场景之间的过渡是怎么处理的？
   - 有没有硬切？硬切的效果好不好？
5. 节奏快慢
   - 哪些地方作者加速了？（一句话带过一段时间）
   - 哪些地方作者减速了？（一个瞬间写了很多字）
   - 快慢的切换是不是合理？

### 六、可复用技巧清单（最终输出）
**核心问题：我怎么能把这些技巧用到自己的书里？**

格式要求：
```
技巧1：[名称]
- 原书怎么用的：[具体描述]
- 我怎么用到自己的书里：[给出具体的应用建议]
- 在提示词里怎么写：[给出一段可以直接放进提示词的指令]
```

至少提取5个可复用技巧。

## 12. 爽点类型枚举

用于 chapter_analysis.jsonl 的 payoffs 字段：

### 压制类
- `被打脸`：当众丢脸、被轻视、被否定
- `被围困`：陷入包围、无路可退
- `被误解`：被冤枉、被污蔑
- `被剥夺`：失去资源、地位、关系

### 释放类
- `打脸兑现`：证明实力、让轻视者吃瘪
- `碾压`：实力压制、无压力获胜
- `获得宝物`：金手指、装备、资源到手
- `升级`：实力/地位提升
- `身份揭示`：隐藏身份公开
- `逆转局面`：从劣势翻盘
- `智商碾压`：用智慧解决困境

### 情绪类
- `护人成功`：保护重要人物
- `关系推进`：与关键人物关系升温
- `真相大白`：误会解除、阴谋曝光

## 13. 章末钩子类型枚举

用于 chapter_analysis.jsonl 的 hook_types 字段：

- `悬念钩`：留下未解之谜
- `反转钩`：剧情意外转折
- `情绪高点`：停在爽/紧张时刻
- `信息差`：读者知道但角色不知道
- `卡在高潮`：动作进行中突然切断
- `事故钩`：新危机突发
- `真相钩`：即将揭露秘密
- `身份钩`：身份即将暴露
- `关系钩`：关系即将变化
