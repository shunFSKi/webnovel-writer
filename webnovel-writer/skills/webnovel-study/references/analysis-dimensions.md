# Analysis Dimensions

## 1. 总目标

分析不是复述剧情，而是回答：
- 这本书怎么抓住读者
- 靠谁抓住读者
- 靠什么节奏一直推下去
- 哪些打法适合迁移

## 2. 输入优先级

分析时优先使用：
1. `study_meta.json`
2. `chapter_index.json`
3. `chapter_source.jsonl`
4. 必要时再参考原书简介或抽样正文

原则：
- 逐章分析以缓存为准，不再重复手动切章
- `confidence` 与 `evidence_basis` 应继承源数据边界

## 3. 逐章分析维度

每章优先回答：
- 这一章解决了什么主问题
- 给了什么反馈
- 抛了什么新问题
- 主角是主动还是被动
- 这一章属于哪个剧情功能

### plot_role 枚举建议
- `开局`
- `承接`
- `升级`
- `对抗`
- `反转`
- `回收`
- `过渡`
- `高潮`
- `低谷`

### hook_types 常见值
- `事故钩`
- `悬念钩`
- `身份钩`
- `关系钩`
- `任务钩`
- `危险钩`
- `真相钩`
- `反差钩`

### payoffs 常见值
- `打脸`
- `资源到手`
- `身份抬升`
- `关系推进`
- `危机暂解`
- `新线索`
- `能力兑现`
- `读者预期兑现`

## 4. `chapter_analysis.jsonl` 最小字段

```json
{
  "chapter_number": 1,
  "chapter_title": "第1章 暴雨夜",
  "summary": "主角在险情初发时被迫接任务，并靠职业判断先做第一轮止损。",
  "plot_role": "开局",
  "hook_types": ["事故钩", "危险钩"],
  "payoffs": ["能力兑现", "危机暂解"],
  "new_question": "这场险情究竟是天灾还是人祸？",
  "characters_entered": ["主角", "直属上官"],
  "conflict_level": 4,
  "style_signals": ["短段快切", "对白带信息"],
  "notes": "基于标题与可读片段推断。",
  "confidence": "medium",
  "evidence_basis": ["title", "excerpt"]
}
```

要求：
- `summary` 控制在 50-120 字
- `style_signals` 只写抽象特征，不抄长句
- `notes` 要诚实说明是否为结构推断

## 5. 剧情结构分析维度

必须看：
- 标题承诺在几章内兑现
- 前三章是否形成小闭环
- 主线是否持续可见
- 中段是否出现失速
- 卷末是否有新钩子接下一段

建议拆成：
- 开篇抓读
- 中段续航
- 后段加速
- 结尾/卷末挂钩

## 6. 人物分析维度

### 主角
- 卖点
- 欲望
- 缺陷
- 先手优势
- 决策模型
- 爽点来源

### 配角
- 功能分工
- 是否能补主角短板
- 是否有单独辨识度
- 是否只在需要时出现

### 反派
- 分层
- 对主角构成哪种压力
- 是否只会提供脸谱化阻力
- 是否形成递进升级

## 7. 节奏分析维度

重点看：
- 每章是否解决一个主问题
- 每章是否留下继续读的理由
- 强反馈是不是来得早
- 过渡章是否也有小推进
- 是否出现连续多章同质化

## 8. 当前项目借鉴分析维度

在 `--compare-current` 下，必须比较：
- 标题承诺兑现速度
- 前三章抓读强度
- 主角主动性
- 爽点反馈速度
- 反派压迫感构建
- 章末留钩具体度
- 每次成果是否换到更大权限

## 9. 退化模式写法要求

### `full_text`
- 可以按正常精读口径输出
- `confidence` 可高于结构推断

### `degraded_structure_only`
- 剧情和节奏仍可正常分析
- 人物/风格/词句必须标 `medium` 或 `low`
- 明确写“基于标题序列、简介、抽样片段推断”

### `blocked`
- 不写逐章分析，不生成完整报告
- 只输出阻断原因

## 10. 不合格分析示例

❌ 不合格：
- 第5章主要讲主角去了哪里，遇到了谁……

✅ 合格：
- 第5章的功能是“把前面埋下的危险转成公开冲突”，主角第一次从被动挨打转成主动抢先手，因此读者会觉得书开始真正起飞。
