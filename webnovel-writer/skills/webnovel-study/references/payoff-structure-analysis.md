# 爽点结构分析指南

## 1. 核心问题

读者为什么追这本书？爽点是怎么铺垫和释放的？

## 2. 分析目标

不是标记"这里有爽点"，而是回答：
- 爽点之前铺了多少章的压？
- 压制的方式是什么？读者有多憋？
- 释放是一次性爆开还是分层给？
- 释放之后有没有收获/反馈环节？
- 这个循环模式能不能迁移？

## 3. 爽点循环结构

### 标准循环
```
压制阶段（铺垫） → 爆发点（爽点释放） → 反馈阶段（收获/验证） → 新压制
```

### 压制方式分类
- 被打脸：当众被轻视、被否定、被嘲讽
- 被围困：陷入包围、无路可逃、被威胁
- 被误解：被冤枉、被污蔑、没人信
- 被剥夺：失去资源、失去地位、失去关系
- 被轻视：实力被看低、存在感被忽视

### 释放方式分类
- 打脸兑现：当众证明实力，让轻视者吃瘪
- 碾压：实力绝对压制，无压力获胜
- 获得宝物：金手指、装备、资源到手
- 身份揭示：隐藏身份公开，众人震惊
- 逆转局面：从绝对劣势翻盘
- 智商碾压：用智慧解决困境
- 护人成功：保护重要人物，展现价值

### 全书爽点循环分析
总结：
- 小循环几章一轮？
- 大循环几章一轮？
- 压制时长分布：1-3章短压制 / 4-10章中压制 / 10章以上长压制
- 释放方式偏好：一次性爆开 vs 分层释放
- 反馈完整性：释放后有没有收获/验证环节

## 6. 不同类型作品的爽点模式

### 技术流/职业流
- 核心爽点：专业能力验证
- 压制方式：被质疑不专业、被外行指挥、被刁难
- 释放方式：用专业知识解决问题，众人震惊
- 反馈环节：获得更大权限/资源/认可

### 权谋流
- 核心爽点：算计成功、局势逆转
- 压制方式：被设局、被孤立、被低估
- 释放方式：局中局、反将一军、借力打力
- 反馈环节：地位提升、对手败退

### 系统流/金手指流
- 核心爽点：系统奖励、能力升级
- 压制方式：遇到困境、资源不足、实力不够
- 释放方式：系统发放奖励/新能力
- 反馈环节：用新能力解决之前无法解决的问题

## 7. 爽点密度评估

- 高密度：每章都有小爽点，3-5章一个大爽点
- 中密度：2-3章一个小爽点，8-10章一个大爽点
- 低密度：5章以上才有一个爽点

**注意**：密度不是越高越好。过度频繁的爽点会疲劳，关键是要让每个爽点都 earned。

## 8. 退化模式要求

- `full_text`：可精确定位爽点位置和释放方式
- `degraded_structure_only`：基于标题推断爽点位置，降低置信度
- `blocked`：不进行爽点分析

## 9. 爽点质量评估

好爽点的标准：
1. 压制足够：读者也跟着憋
2. 释放充分：不是点到为止，而是彻底解决
3. 有反馈：释放后能获得实际收获
4. 开新局：释放后开启新的故事线

坏爽点的特征：
1. 压制不足：读者没感受到憋屈
2. 释放太弱：解决得不彻底
3. 没反馈：爽完就完了，什么都没得到
4. 断链：释放后接不上新剧情

## 10. 对当前项目的借鉴

在 `07_对当前项目建议.md` 中分析：

1. 当前项目的爽点密度如何？
2. 压制方式是否足够？
3. 释放是否充分？
4. 反馈环节是否完整？
5. 最值得调整的是哪部分？
