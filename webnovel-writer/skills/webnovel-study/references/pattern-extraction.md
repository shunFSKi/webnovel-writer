# Pattern Extraction

## 1. 模式的定义

模式不是“原文某一句”，而是可反复复用的写作打法。

一个好模式通常满足：
- 重复出现 2 次以上，或一次但极关键
- 能解释读者为什么会追
- 能迁移到其他项目，而不是只适配这一本书

## 2. 常见 pattern_type

- `hook`：开篇钩、章末钩、事故钩、真相钩
- `pacing`：前三章小闭环、中段反转、卷末挂钩
- `dialogue`：对白推进、话风区分、信息型对白
- `payoff`：打脸兑现、资源到账、身份抬升
- `emotion`：关系拉扯、护人、嘴硬心软、压后再放
- `structure`：主线+副线并跑、双任务驱动、反派分层升级

## 3. 每条模式必须回答

- 这是什么打法
- 它在哪些章节出现
- 它为什么有效
- 它迁移到别的书时要怎么改
- 它最大的风险是什么

## 4. learnability 分层

### 可直接学
- 不依赖特定设定外壳
- 迁移成本低
- 对多数网文项目都成立

### 需改造
- 结构可学，但必须换成当前项目自己的题材壳
- 若直接套用，会显得违和

### 不建议学
- 强绑定原书特殊世界观/主角特性
- 前期有效，后期容易模板化
- 有明显副作用

## 5. score 建议口径（1-10）

- 9-10：强可学，高收益，低副作用
- 7-8：值得学，但有适配条件
- 5-6：可作参考，不建议直接迁移
- 1-4：只适合理解，不建议学习

## 6. JSON 最小结构

```json
{
  "pattern_type": "hook",
  "name": "事故兑现开篇",
  "description": "标题承诺在第一章末落地",
  "evidence_range": "第1-3章",
  "transfer_rule": "适合事故型男频开篇",
  "risk": "若后续反馈跟不上会掉速",
  "adaptation_note": "迁移到治河文时改成堤身黑浆异象",
  "score": 9,
  "learnability": "可直接学",
  "confidence": "medium",
  "evidence_basis": ["title", "excerpt"]
}
```

说明：
- `confidence` 建议继承自相关章节的分析可信度
- `evidence_basis` 建议写明是来自 `title`、`excerpt` 还是更广的结构归纳

## 7. transfer_rule 示例

✅ 好：
- `适合事故型男频开篇，迁移时要保证主角当场接任务，而不是只目击事故。`
- `适合技术流/职业流，把“专业动作 -> 当场反馈 -> 群体变脸”串成一条线。`

❌ 差：
- `这个很好，可以学。`

## 8. adaptation_note 示例

✅ 好：
- `迁移到当前治河题材时，可把“身份事故”替换成“验收合格堤身突发黑浆”，但不要照搬原书关系冲突。`

## 9. 风险说明示例

- 同一事故钩连续使用会疲劳
- 过度依赖嘴炮打脸会削弱真实感
- 关系拉扯如果没有身份/资源变化支撑，会空转
- 纯短句轰炸在历史文里容易丢质感

## 10. 退化模式要求

- 若 `analysis_mode=degraded_structure_only`，仍可提炼模式，但要优先提炼结构/节奏/钩子类模式。
- 人物细部、措辞细部类模式必须降低置信度，不能伪装成全文高置信规律。
- 若 `analysis_mode=blocked`，不应生成 `06_可复用模式.json`。

## 11. 最终数量控制

建议：
- 整本书总模式数：5-20 条
- 核心模式：3-8 条
- 宁少勿滥，避免把普通桥段也包装成模式
